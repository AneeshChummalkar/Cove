# Notifications

Notifications keep users informed about task completion, background agents, security events, and required approvals.

## Notification Types

- Task completed
- Task failed
- Agent needs approval
- Agent paused
- Agent completed
- New device login
- Security setting changed
- Kill switch activated

## Principles

- Notify only when useful
- Include the acting agent or task name
- Include the next required user decision
- Route sensitive confirmations to overlays
- Record notification-triggering events in the audit log
